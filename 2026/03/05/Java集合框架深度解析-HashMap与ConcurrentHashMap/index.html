Java 集合框架深度解析 - HashMap 与 ConcurrentHashMap

Java 集合框架深度解析 - HashMap 与 ConcurrentHashMap

一、核心知识点

1.1 HashMap 数据结构

JDK 1.8 之前：数组 + 链表
JDK 1.8 之后：数组 + 链表 + 红黑树

为什么转红黑树？

  • 链表查找：O(n)
  • 红黑树查找：O(log n)
  • 提升极端情况下的性能

1.2 HashMap 核心参数

为什么负载因子是 0.75？

  • 空间和时间权衡
  • 太小：浪费空间
  • 太大：哈希冲突多，性能下降
  • 0.75 是泊松分布的最佳值

1.3 哈希计算

为什么这样计算？

  • 减少哈希冲突
  • 让高位参与运算
  • 数组长度通常是 2 的幂，低位决定位置
  • 高位异或后，让高位特征也影响位置

1.4 扩容机制

触发条件：size > capacity * loadFactor

扩容过程：

  1. 容量翻倍（newCap = oldCap << 1）
  2. 创建新数组
  3. 重新计算每个元素的位置
  4. 迁移数据

1.5 ConcurrentHashMap 实现原理

JDK 1.7：分段锁（Segment）

JDK 1.8：CAS + synchronized

为什么 JDK 1.8 改进？

  • 分段锁粒度粗（整个 Segment）
  • 新方案锁粒度细（单个节点）
  • 减少内存占用（不需要 16 个 Segment）
  • 并发度更高

1.6 ConcurrentHashMap 核心操作

put 操作流程：

  1. 计算 hash 值
  2. 如果数组为空，初始化
  3. 如果位置为空，CAS 插入
  4. 如果正在扩容，帮助扩容
  5. 如果有节点，synchronized 锁住头节点
  6. 插入链表或红黑树
  7. 检查是否需要转红黑树
  8. 更新计数，检查是否需要扩容

get 操作：

  • 无锁
  • Node 的 val 和 next 是 volatile 的
  • 保证可见性

二、面试高频题

Q1: HashMap 为什么线程不安全？

A1:

1. JDK 1.7 死循环

2. JDK 1.8 数据覆盖

3. size 计数不准确

Q2: HashMap 的扩容为什么是 2 的幂次方？

A2:

原因：

  1. 位运算效率高于取模
  2. (n - 1) & hash 等价于 hash % n（当 n 是 2 的幂）
  3. 元素分布更均匀

Q3: ConcurrentHashMap 如何保证线程安全？

A3:

1. 使用 volatile 保证可见性

2. 使用 CAS 操作

3. 使用 synchronized 锁

4. 使用 sizeCtl 控制扩容

Q4: HashMap 和 ConcurrentHashMap 性能对比？

A4:

操作 HashMap ConcurrentHashMap 说明
单线程 put 快 稍慢（额外检查） HashMap 无同步开销
多线程 put 不安全 安全 HashMap 需要外部同步
get O(1) O(1) 性能接近
扩容 快 慢（并发扩容） ConcurrentHashMap 扩容复杂

结论：

  • 单线程用 HashMap
  • 多线程用 ConcurrentHashMap
  • 不要用 Collections.synchronizedMap（性能差）

Q5: 如何解决 HashMap 哈希冲突？

A5:

1. 链地址法（Java 采用）

2. 开放地址法

3. 再哈希法

4. 建立公共溢出区

四、延伸思考

4.1 为什么 JDK 1.8 不用 ReentrantLock？

原因：

  1. 内存占用：每个 Segment 都是 ReentrantLock，内存开销大
  2. 并发度：Segment 数量固定（默认 16），并发度受限
  3. 性能：synchronized 在 JDK 1.6 后优化很好
  4. 锁粒度：新方案锁单个节点，粒度更细

4.2 红黑树 vs 跳表

特性 红黑树 跳表
查找 O(log n) O(log n)
插入 O(log n) O(log n)
实现 复杂 简单
并发 难 易（ConcurrentSkipListMap）
内存 较少 较多（多层索引）

ConcurrentSkipListMap：

  • 基于跳表实现
  • 支持排序
  • 无锁算法
  • 适合需要排序的场景

4.3 HashMap vs TreeMap vs LinkedHashMap

特性 HashMap TreeMap LinkedHashMap
顺序 无序 排序（红黑树） 插入顺序或访问顺序
查找 O(1) O(log n) O(1)
空间 最小 较大 中等
适用场景 快速查找 需要排序 需要保持顺序
