Q1: synchronized 和 Lock 的区别？

A1: 推荐回答

特性 synchronized Lock
实现 JVM 层面 Java API
释放锁 自动（代码块结束） 手动（finally）

Q3: 线程池的 7 个参数是什么？

A3: 推荐回答

参数详解：

  1. corePoolSize：核心线程数，即使空闲也不回收
  2. maximumPoolSize：最大线程数，核心 + 临时线程
  3. keepAliveTime：临时线程空闲存活时间
  4. unit：时间单位
  5. workQueue：等待队列（ArrayBlockingQueue、LinkedBlockingQueue、SynchronousQueue）
  6. threadFactory：创建线程的工厂
  7. handler：拒绝策略（AbortPolicy、CallerRunsPolicy、DiscardPolicy、DiscardOldestPolicy）

执行流程：

拒绝策略：

  • AbortPolicy（默认）：抛出异常
  • CallerRunsPolicy：调用者线程执行
  • DiscardPolicy：直接丢弃
  • DiscardOldestPolicy：丢弃最老的任务

四、延伸思考

4.1 synchronized vs ReentrantLock 性能对比

JDK 1.6 之前：

  • synchronized 性能差（重量级锁）
  • ReentrantLock 性能好

JDK 1.6+：

  • synchronized 优化（偏向锁、轻量级锁）
  • 性能差距很小

选择建议：

  • 简单场景：synchronized
  • 需要高级功能：ReentrantLock
  • 高竞争：ReentrantLock 略优
