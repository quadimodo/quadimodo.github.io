环境信息

  • 系统: Linux (Docker)
  • OpenVAS 版本: Greenbone Community Edition 24.x
  • 部署方式: Docker Compose

问题一：Web UI 显示 404

现象

访问 http://IP:9392/ 返回 404 错误

原因分析

gsad 容器启动参数是 --http-only --api-only -f，这是 API-only 模式，不提供 Web UI 静态页面，只提供 GMP API 接口。

解决方案

需要添加 gsa 容器来生成静态文件，然后用 nginx 提供 Web UI。

问题二：无法创建扫描任务（扫描配置不可用）

现象

Web UI 可以登录，但创建扫描任务时提示”扫描配置不可用”或配置下拉菜单为空

诊断步骤

  1. 检查 configs 表:

如果返回 0 行，说明扫描配置没有加载。

  1. 检查 NVT 数据:

如果返回 0，说明 VT（漏洞测试脚本）没有同步到数据库。

  1. 检查 gvmd 日志:

关键日志信息：

  • No SCAP database found - SCAP 数据库未初始化
  • OSP service has different VT status - VT 数据需要同步
  • Failed to open lock file - 权限问题

常见原因及解决方案

原因 A：SCAP 数据库未同步

症状: 日志显示 No SCAP database found 或 rebuilding SCAP db from scratch

解决: 等待 SCAP 数据同步完成（约 30-60 分钟）

同步完成标志：

  • Updating SCAP info succeeded
  • Updating CVSS scores and CVE counts for CPEs 完成

原因 B：VT 同步卡住

问题四：信号量权限错误

现象

执行 gvmd 命令时出现：

原因

gvmd 容器以 root 运行，但 gvmd 守护进程以 gvmd 用户运行，命令行工具与守护进程竞争信号量。

解决方案

方案 1: 以 gvmd 用户身份执行命令

方案 2: 通过运行中的守护进程执行（不推荐直接运行 gvmd 命令）

完整的状态检查清单

关键日志关键词

关键词 含义 处理建议
No SCAP database found SCAP 数据库未初始化 等待自动同步
rebuilding SCAP db from scratch 正在重建 SCAP 数据库 等待完成
Updating SCAP info succeeded SCAP 同步完成 ✅ 正常
OSP service has different VT status VT 需要同步 等待或手动触发
Failed to open lock file 权限问题 修复目录权限
lock file already exists 锁冲突 删除锁文件重启
configs has been created 配置已加载 ✅ 正常
Finished loading VTs VT 加载完成 ✅ 正常

预防措施

  1. 首次部署后耐心等待

    • SCAP 数据同步需要 30-60 分钟
    • VT 同步需要 5-10 分钟
    • 不要频繁重启容器

  2. 定期检查日志

  3. 数据卷备份
