JDK 中的应用：

二、结构型模式

2.1 适配器模式（Adapter）

作用：将一个类的接口转换成客户端期望的另一个接口

JDK 中的应用：

  • java.io.InputStreamReader（字节流 → 字符流）
  • java.io.OutputStreamWriter
  • java.util.Arrays.asList()

2.2 装饰器模式（Decorator）

作用：动态地给对象添加额外功能

JDK 中的应用：

  • java.io 包中的各种流
  • Collections.synchronizedList()
  • Collections.unmodifiableList()

2.3 代理模式（Proxy）

作用：控制对对象的访问

JDK 中的应用：

  • java.lang.reflect.Proxy
  • Spring AOP
  • MyBatis Mapper 代理

2.4 外观模式（Facade）

作用：为子系统提供一个统一的高层接口

JDK 中的应用：

  • java.util.Collections
  • javax.faces.context.FacesContext

2.5 组合模式（Composite）

作用：将对象组合成树形结构以表示”部分-整体”的层次结构

三、行为型模式

3.1 策略模式（Strategy）

作用：定义一系列算法，让它们可以互相替换

JDK 中的应用：

  • Comparator 接口
  • ThreadPoolExecutor 的拒绝策略

3.2 模板方法模式（Template Method）

作用：定义算法骨架，将某些步骤延迟到子类实现

JDK 中的应用：

  • AbstractList、AbstractSet
  • HttpServlet 的 service() 方法
  • Spring JdbcTemplate

3.3 观察者模式（Observer）

作用：定义对象间的一对多依赖，当对象状态改变时通知所有依赖

JDK 中的应用：

  • java.util.Observer / Observable（已废弃）
  • java.util.EventListener
  • Spring 事件机制

3.4 责任链模式（Chain of Responsibility）

作用：将请求沿着处理链传递，直到有处理者处理

JDK 中的应用：

  • Servlet Filter
  • Spring Interceptor
  • Netty Pipeline

3.5 命令模式（Command）

作用：将请求封装为对象，支持撤销、重做等操作

JDK 中的应用：

  • Runnable 接口
  • Swing Action

四、面试高频题

Q1: 单例模式有几种实现方式？

A1: 推荐回答

方式 线程安全 延迟加载 防止反射 防止序列化
饿汉式 ✓ ✗ ✗ ✗
双重检查 ✓ ✓ ✗ ✗
静态内部类 ✓ ✓ ✗ ✗
枚举 ✓ ✗ ✓ ✓

推荐：枚举单例（最简单、最安全）

Q2: 工厂方法和抽象工厂的区别？

A2: 推荐回答

特性 工厂方法 抽象工厂
产品数量 一种 多种（产品族）
工厂数量 每个产品一个工厂 每个产品族一个工厂
扩展 新增产品容易 新增产品族容易

Q3: 代理模式和装饰器模式的区别？

A3: 推荐回答

特性 代理模式 装饰器模式
目的 控制访问 增强功能
创建对象 代理控制 客户端控制
关系 代理和真实对象相同接口 装饰器和组件相同接口

关键区别：

  • 代理：强调”控制”
  • 装饰器：强调”增强”

Q4: Spring 中用了哪些设计模式？

A4: 推荐回答

模式 Spring 应用
工厂模式 BeanFactory、ApplicationContext
单例模式 Bean 默认单例
代理模式 AOP、事务
模板方法 JdbcTemplate、RestTemplate
策略模式 Resource 加载
观察者模式 ApplicationEvent
适配器模式 HandlerAdapter
装饰器模式 BeanWrapper
责任链模式 Filter、Interceptor

Q5: 什么时候用策略模式？

适用场景：

  • 多种算法实现同一功能
  • 需要动态切换算法
  • 避免大量 if-else
